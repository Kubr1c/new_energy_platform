# 系统负荷预测模型改进与调优方案

## 一、背景与挑战

在能源管理系统中，准确的负荷预测对于能源调度、成本控制和设备运维至关重要。传统的预测模型往往存在预测值与真实值差距较大的问题，特别是在负荷高峰期和异常天气条件下。

为了解决这一问题，系统地改进了预测模型，从数据质量、特征工程、模型架构到训练策略等多个维度进行了全面优化。

## 二、预测模型体系

构建了一个包含六个不同算法的预测模型体系，从简单到复杂，覆盖了当前主流的时间序列预测技术。

### 1. 标准 LSTM 模型

**原理**：LSTM（长短期记忆网络）是一种专门处理时间序列数据的循环神经网络。它通过门控机制（输入门、遗忘门、输出门）有效解决了传统RNN的梯度消失问题，能够捕捉长期依赖关系。

**架构**：
- 双层LSTM结构
- 输入维度：包含时间特征和外部特征
- 输出维度：预测未来3小时的负荷值

**核心实现**：
```python
class StandardLSTM(nn.Module):
    def __init__(self, input_size, hidden_size1=64, hidden_size2=32, output_size=1, dropout=0.2):
        super(StandardLSTM, self).__init__()
        # 第一层 LSTM：处理输入序列，提取初级时序特征
        self.lstm1 = nn.LSTM(
            input_size=input_size,  # 输入特征维度
            hidden_size=hidden_size1,  # 隐藏层维度
            batch_first=True,  # 输入格式为 (batch, seq_len, features)
            dropout=dropout  # 防止过拟合
        )
        # 第二层 LSTM：进一步提炼时序特征
        self.lstm2 = nn.LSTM(
            input_size=hidden_size1,  # 输入特征维度，对应第一层 LSTM 的输出
            hidden_size=hidden_size2,  # 隐藏层维度
            batch_first=True,
            dropout=dropout
        )
        # 全连接层：将 LSTM 输出映射到中间维度
        self.fc = nn.Linear(hidden_size2, 16)  # 中间维度设为 16
        # 输出层：将中间维度映射到最终预测目标
        self.out = nn.Linear(16, output_size)
        # Dropout 层：防止过拟合
        self.dropout = nn.Dropout(dropout)
```

**调试思路**：
- 观察训练曲线，确保模型收敛
- 调整隐藏层大小（通常128-256之间）
- 检查梯度范数，防止梯度爆炸
- 验证集性能监控，避免过拟合

### 2. GRU 模型

**原理**：GRU（门控循环单元）是LSTM的轻量级替代方案，它将LSTM的三个门简化为两个门（更新门和重置门），在保持性能的同时减少了参数量。

**优势**：
- 参数更少（约15K，比LSTM少25%）
- 训练速度更快
- 内存占用更小

**调试思路**：
- 与LSTM对比训练速度和准确率
- 调整门控单元的激活函数
- 测试不同的序列长度对性能的影响

### 3. Attention-LSTM 模型

**原理**：在LSTM的基础上引入注意力机制，使模型能够自动聚焦于对预测最关键的历史时段，提高预测精度。

**架构**：
- 双层LSTM
- Bahdanau注意力机制
- 注意力权重可视化

**核心实现**：
```python
class BahdanauAttention(nn.Module):
    def __init__(self, hidden_size, query_size, attn_size=None):
        super().__init__()
        if attn_size is None:
            attn_size = hidden_size // 2
        # 定义注意力机制的线性变换层
        self.W_h = nn.Linear(hidden_size, attn_size, bias=False)  # 对序列隐状态的线性变换
        self.W_q = nn.Linear(query_size,  attn_size, bias=False)  # 对 query 的线性变换
        self.v   = nn.Linear(attn_size, 1, bias=False)  # 计算注意力得分的线性变换

    def forward(self, encoder_outputs, query):
        # 计算注意力能量值
        energy = torch.tanh(
            self.W_h(encoder_outputs) +             # (batch, seq_len, attn_size)
            self.W_q(query).unsqueeze(1)            # (batch, 1, attn_size)，扩展维度
        )
        # 计算注意力得分
        scores = self.v(energy)                    # (batch, seq_len, 1)
        # 归一化注意力得分得到权重
        weights = torch.softmax(scores, dim=1)      # (batch, seq_len, 1)
        # 计算加权上下文向量
        context = (weights * encoder_outputs).sum(dim=1)  # (batch, hidden_size)
        return context, weights
```

**调试思路**：
- 分析注意力权重分布，验证模型是否正确关注关键时段
- 调整注意力头数和注意力维度
- 测试不同的注意力计算方式

### 4. CNN-LSTM 混合模型

**原理**：结合CNN的局部特征提取能力和LSTM的时序建模能力，先通过一维卷积提取负荷数据的局部模式，再用LSTM捕捉时序依赖。

**架构**：
- 4层一维卷积网络
- 跳跃连接（Residual Connection）
- 双层LSTM

**核心实现**：
```python
class CNNLSTM(nn.Module):
    def __init__(self, input_size, hidden_size1=128, hidden_size2=64, output_size=1, dropout=0.3):
        super(CNNLSTM, self).__init__()
        # CNN 特征提取层
        # 第一层卷积：从输入特征维度映射到 64 通道，使用 5x5 卷积核
        self.conv1 = nn.Conv1d(
            in_channels=input_size,  # 输入通道数等于特征维度
            out_channels=64,  # 输出通道数
            kernel_size=5,  # 卷积核大小，捕捉 5 个时间步的局部特征
            padding=2  # 填充，保持序列长度不变
        )
        # 第二层卷积：进一步提取特征
        self.conv2 = nn.Conv1d(
            in_channels=64,  # 输入通道数
            out_channels=128,  # 输出通道数
            kernel_size=3,  # 卷积核大小
            padding=1  # 填充
        )
        # 第三层卷积：更深层特征提取
        self.conv3 = nn.Conv1d(
            in_channels=128,  # 输入通道数
            out_channels=256,  # 输出通道数
            kernel_size=3,  # 卷积核大小
            padding=1  # 填充
        )
        # 第四层卷积：最终特征提取
        self.conv4 = nn.Conv1d(
            in_channels=256,  # 输入通道数
            out_channels=128,  # 输出通道数
            kernel_size=3,  # 卷积核大小
            padding=1  # 填充
        )
```

**调试思路**：
- 调整卷积核大小和数量
- 测试不同的池化策略
- 验证卷积层输出的特征图质量

### 5. Transformer 模型

**原理**：基于自注意力机制的纯注意力架构，能够并行处理序列数据，捕捉全局依赖关系，是当前学术前沿的时序预测方案。

**架构**：
- 多头自注意力机制
- 位置编码（Positional Encoding）
- 前向神经网络

**核心实现**：
```python
class TimeSeriesTransformer(nn.Module):
    def __init__(self, input_size, hidden_size1=64, hidden_size2=32, output_size=1, dropout=0.2):
        super(TimeSeriesTransformer, self).__init__()
        d_model = hidden_size1  # Transformer 内部维度
        # 输入投影层：将原始特征维度映射到 d_model 维
        self.input_projection = nn.Linear(input_size, d_model)
        # 位置编码
        self.positional_encoding = PositionalEncoding(d_model, dropout=dropout)
        # Transformer Encoder
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=d_model,           # 模型维度
            nhead=4,                  # 4 头注意力
            dim_feedforward=128,       # 前馈网络隐藏层维度
            dropout=dropout,           # Dropout 概率
            batch_first=True           # 输入格式为 (batch, seq_len, features)
        )
        # 堆叠多个编码器层
        self.transformer_encoder = nn.TransformerEncoder(encoder_layer, num_layers=2)  # 2 层编码器
```

**调试思路**：
- 调整注意力头数（通常4-8个）

- 控制模型复杂度，防止过拟合

### 6. Temporal Fusion Transformer (TFT) 模型

**原理**：这是引入的最先进模型，结合了Transformer的注意力机制和门控循环单元，专门设计用于时间序列预测，能够同时处理静态特征和时间特征。

**核心创新**：
- 时间注意力机制：关注不同时间步的重要性
- 特征选择网络：自动学习特征重要性
- 门控融合层：有效整合静态和动态信息

**核心实现**：
```python
class GatedResidualNetwork(nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim, dropout=0.1):
        super(GatedResidualNetwork, self).__init__()
        # 第一层全连接：将输入维度映射到隐藏维度
        self.fc1 = nn.Linear(input_dim, hidden_dim)
        # 门控线性单元：实现特征选择
        self.glu = GatedLinearUnit(hidden_dim, hidden_dim)
        # 第二层全连接：将隐藏维度映射到输出维度
        self.fc2 = nn.Linear(hidden_dim, output_dim)
        # Dropout层：在训练时随机断开部分连接，防止过拟合
        self.dropout = nn.Dropout(dropout)
        # 层归一化：稳定训练过程，加速收敛
        self.layer_norm = nn.LayerNorm(output_dim)
        # 残差连接：输入和输出维度不同时，需要线性变换来匹配
        self.residual = nn.Linear(input_dim, output_dim) if input_dim != output_dim else None
```

## 三、调优算法详解

为了进一步提升模型性能，实现了三个关键调优算法：

### 1. 早停机制 (EarlyStopping)

**原理**：早停是一种防止过拟合的正则化技术，通过监控验证集性能，当模型性能连续多次没有改善时自动停止训练。

**工作流程**：
1. 训练过程中持续监控验证集损失
2. 记录历史最佳损失值和对应的模型权重
3. 如果验证损失连续patience个epoch没有改善，则停止训练
4. 训练结束后，加载历史最佳权重

**核心实现**：
```python
class EarlyStopping:
    def __init__(self, patience=10, delta=0, path='checkpoint.pt'):
        self.patience = patience  # 早停耐心值
        self.delta = delta  # 最小改善阈值
        self.path = path  # 最佳模型保存路径
        self.counter = 0  # 连续未改善的epoch计数器
        self.best_score = None  # 历史最佳得分（负的损失值）
        self.early_stop = False  # 是否触发早停
        self.val_loss_min = np.Inf  # 历史最低验证损失

    def __call__(self, val_loss, model):
        # 将损失转换为得分（损失越低，得分越高）
        score = -val_loss
        # 第一次调用：记录初始值
        if self.best_score is None:
            self.best_score = score  # 记录当前得分为最佳
            self.save_checkpoint(val_loss, model)  # 保存当前模型
        # 得分没有显著改善
        elif score < self.best_score + self.delta:
            self.counter += 1  # 未改善计数器加1
            if self.counter >= self.patience:
                # 连续patience个epoch未改善，触发早停
                self.early_stop = True
        # 得分有显著改善
        else:
            self.best_score = score  # 更新最佳得分
            self.save_checkpoint(val_loss, model)  # 保存当前最佳模型
            self.counter = 0  # 重置未改善计数器
        return self.early_stop
```

**调整思路**：
- patience参数：通常设置为5-20之间，太小容易过早停止，太大可能过拟合
- delta参数：最小改善阈值，通常设为0
- 保存最佳模型权重，确保最终使用的是性能最好的模型

### 2. 学习率调度器 (LearningRateScheduler)

**原理**：学习率是深度学习中最重要的超参数之一，合适的学习率调整策略可以加速收敛，提高模型性能。

**支持的调度策略**：
1. **余弦退火**：学习率按照余弦函数曲线周期性变化，适合训练后期精细调整
2. **阶梯衰减**：每隔固定epoch数，将学习率降低一定比例，简单有效
3. **Reduce on Plateau**：监控验证集损失，当损失停止下降时自动降低学习率，本项目使用此策略

**核心实现**：
```python
class LearningRateScheduler:
    def __init__(self, optimizer, mode='cosine', initial_lr=0.001, min_lr=1e-6, patience=5):
        self.optimizer = optimizer  # 保存优化器引用
        self.mode = mode  # 调度模式
        self.initial_lr = initial_lr  # 初始学习率
        self.min_lr = min_lr  # 最小学习率下限
        self.patience = patience  # Plateau耐心值

        # 根据模式创建对应的调度器
        if mode == 'cosine':
            # 余弦退火：学习率先上升再下降（或持续下降）
            self.scheduler = optim.lr_scheduler.CosineAnnealingLR(
                optimizer, T_max=100, eta_min=min_lr
            )
        elif mode == 'step':
            # 阶梯衰减：每10个epoch学习率减半
            self.scheduler = optim.lr_scheduler.StepLR(
                optimizer, step_size=10, gamma=0.5
            )
        elif mode == 'reduce_on_plateau':
            # 验证损失 Plateau 时降低学习率
            self.scheduler = optim.lr_scheduler.ReduceLROnPlateau(
                optimizer, mode='min', factor=0.5, patience=patience, min_lr=min_lr
            )
```

**调整思路**：
- 初始学习率：通常从0.001开始
- 最小学习率：设置为1e-6，防止学习率过低
- 衰减因子：通常设置为0.5，每次降低一半
- 耐心值：监控验证损失的epoch数，通常为5-10

### 3. 自定义损失函数

**原理**：标准损失函数对所有样本一视同仁，但实际应用中，我们可能希望对某些样本（如高峰时段）给予更多关注。

**实现的损失函数**：
1. **加权MSE损失**：对高峰时段赋予更高权重，提高峰值预测准确性
2. **Huber损失**：对异常值更鲁棒，减少极端值影响
3. **组合损失**：结合多种损失函数，平衡不同误差指标

**核心实现**：
```python
class WeightedMSELoss(nn.Module):
    def __init__(self, peak_hours=None, peak_weight=2.0):
        super(WeightedMSELoss, self).__init__()
        # 设置默认高峰时段（早晚高峰）
        self.peak_hours = peak_hours or [7, 8, 18, 19, 20]
        self.peak_weight = peak_weight  # 高峰时段权重倍数
        # 标准MSE损失（用于非加权情况）
        self.mse_loss = nn.MSELoss()

    def forward(self, pred, target, hour_indices=None):
        # 如果没有提供小时索引，使用标准MSE
        if hour_indices is None:
            return self.mse_loss(pred, target)

        # 创建权重矩阵
        weights = torch.ones_like(pred)

        # 为高峰时段分配更高权重
        for hour in self.peak_hours:
            # 找出属于高峰时段的样本
            mask = (hour_indices == hour)
            # 分配高峰权重
            weights[mask] = self.peak_weight

        # 计算加权MSE
        loss = (weights * (pred - target) ** 2).mean()

        return loss
```

**调整思路**：
- 加权MSE：根据实际负荷曲线确定高峰时段（通常是7-8点和18-20点）
- Huber损失：delta参数通常设置为1.0，平衡MSE和MAE的特性
- 组合损失：根据具体场景调整不同损失函数的权重
