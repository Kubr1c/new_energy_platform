# 新能源储能调度系统技术开发文档

## 1. 项目目标与范围

本项目实现了“数据管理 -> 功率预测 -> 优化调度 -> 历史分析 -> 用户权限管理”的完整闭环，主要面向新能源侧（风电/光伏）与储能联合运行场景。

核心能力：
- 新能源与负荷数据上传、查询、统计
- 基于 Attention-LSTM 的风电/光伏/负荷预测
- 基于 AWPSO 的储能多目标优化调度
- 调度历史对比与模拟执行评估
- JWT 鉴权与管理员权限控制

## 2. 总体架构

### 2.1 技术栈

- 后端：Flask + SQLAlchemy + PyTorch + NumPy + Pandas
- 前端：Vue 3 + Vuex + Vue Router + Element Plus + ECharts (`vue-echarts`)
- 数据库：MySQL

### 2.2 分层结构

- **前端展示层**：`frontend/src/views/*`
- **后端接口层**：`backend/routes/*`
- **业务模型层**：`backend/models/*`、`backend/optimization/*`
- **持久化层**：`backend/models/database.py`
- **配置与启动层**：`backend/config.py`、`backend/app.py`

## 3. 后端模块实现

## 3.1 应用启动与基础配置

文件：`backend/app.py`、`backend/config.py`

- `create_app()` 完成 Flask 初始化、CORS 配置、数据库初始化、蓝图注册
- 支持前端构建产物静态托管（`static_folder=../frontend/dist`）
- 启动时可自动 `db.create_all()` 建表
- 全局配置包含：
  - `SECRET_KEY`
  - `SQLALCHEMY_DATABASE_URI`
  - 模型/Scaler 文件路径
  - 储能参数默认值（容量、功率、效率、SOC 上下限、电网上限）

> 注意：`app.py` 中错误处理函数使用了 `jsonify`，需要确保已导入。

## 3.2 数据库模型

文件：`backend/models/database.py`

- `User`：用户表（用户名、密码哈希、角色、状态、最后登录）
- `NewEnergyData`：原始时序数据（时间戳、风/光/负荷、气象）
- `PredictResult`：预测结果（类型、起始时间、步长、预测值、MAPE）
- `DispatchResult`：调度结果（充放电计划、SOC 曲线、弃电率、成本）

数据类型说明：
- 时序计划类字段（充放电、SOC）使用 JSON 存储，便于直接返回前端绘图
- 关键时间字段统一为 `datetime`，接口返回 ISO 格式字符串

## 3.3 认证与用户管理模块

文件：`backend/routes/auth.py`

### 登录鉴权
- `POST /api/auth/login`：用户名密码登录，返回 JWT
- `token_required` 装饰器：
  - 从 `Authorization: Bearer <token>` 提取 token
  - 校验过期、合法性、用户存在性

### 用户管理（管理员）
- `GET /api/auth/users`
- `PUT /api/auth/users/<id>`
- `DELETE /api/auth/users/<id>`
- `POST /api/auth/users/<id>/reset_password`

权限策略：
- `admin` 才能查看/修改/删除/重置其他用户
- 禁止删除当前登录用户自身

## 3.4 数据管理模块

文件：`backend/routes/data.py`

### 上传
- `POST /api/data/upload`
- 校验 CSV 必备列：
  - `timestamp`, `wind_power`, `pv_power`, `load`, `temperature`, `irradiance`, `wind_speed`
- 根据 `timestamp` 做 upsert（存在则更新，不存在则插入）

### 查询与统计
- `GET /api/data/query`：按时间范围+条数查询
- `GET /api/data/latest`：获取最近 24 小时（允许不足 24 条）
- `GET /api/data/statistics`：返回总量、时间范围、均值/峰值统计

## 3.5 预测模块

文件：`backend/routes/predict.py`、`backend/models/predict.py`、`backend/models/attention_lstm.py`

### 模型结构（Attention-LSTM）
- 两层 LSTM：
  - `lstm1(input_size -> hidden1)`
  - `lstm2(hidden1 -> hidden2)`
- 在 `lstm1` 输出上做注意力加权，提取上下文向量
- 将注意力上下文与 `lstm2` 最后时刻隐藏状态拼接后过全连接输出
- 支持多输出（风电/光伏/负荷）

### 推理流程
- `Predictor` 加载：
  - 模型权重：`backend/models_saved/attention_lstm.pth`
  - 归一化器：`data/scaler.pkl`
- 输入最近 `input_len=24` 时序特征
- 归一化 -> 模型推理 -> 按目标字段反归一化 -> 返回字典结果

### 接口实现
- `POST /api/predict/single`：单步预测
- `POST /api/predict/batch`：滚动预测未来 24 步
- `GET /api/predict/history`：预测历史
- `POST /api/predict/evaluate`：根据真实值计算 MAPE
- `DELETE /api/predict/<id>`、`POST /api/predict/batch_delete`

### 关键增强（已实现）
- 数据不足 24 条时自动补齐（用最早记录向前补）
- 兼容空 JSON 请求体
- numpy 输出统一序列化为 Python 标量/数组，避免 JSON 与前端图表异常

## 3.6 调度优化模块

文件：`backend/routes/dispatch.py`、`backend/optimization/dispatch_model.py`、`backend/optimization/awpso.py`、`backend/optimization/solver.py`

### 目标与约束建模

`DispatchModel` 决策变量：
- `Pc(24)`：充电功率
- `Pd(24)`：放电功率
- `abandon(24)`：弃风弃光功率

优化目标：
1. 运行成本（购售电+运维）
2. 弃风弃光率
3. 寿命损耗（基于 DOD 曲线近似）

约束：
- SOC 动态与上下限
- 充放电功率上限
- 电网购售功率上限
- 弃电不超过可再生出力

### 求解器实现

`AWPSO` 采用惩罚函数方式处理约束，内部使用固定权重组合计算适应度。

`solver.py` 做了接口兼容：
- 预测字段支持 `wind/pv/load` 与 `wind_power/pv_power/load`
- 储能参数支持前端键名 `capacity/power` 自动映射到 `ess_capacity/ess_power`

### 调度接口
- `POST /api/dispatch/exec`
- `POST /api/dispatch/multi_objective`
- `GET /api/dispatch/history`
- `GET /api/dispatch/detail/<id>`
- `POST /api/dispatch/simulate`
- `GET /api/dispatch/statistics`

## 4. 训练模块实现

文件：`backend/models/train.py`、`backend/preprocessing/data_utils.py`

训练流程：
1. 读取训练 CSV（支持自动路径解析）
2. 缺失值处理（插值 + 前后填充）
3. 异常值处理（3σ + 中位数替换）
4. MinMax 归一化并保存 scaler
5. 滑窗构造监督样本：
   - `X`: 最近 `input_len` 个时间步的 6 维特征
   - `y`: 未来 `output_len` 步的 3 维目标
6. 训练/验证切分，MSE 损失，Adam 优化，学习率调度，早停
7. 保存最优权重到 `backend/models_saved/attention_lstm.pth`

已做兼容：
- `python -m models.train` 与 `python models/train.py` 双模式导入
- 训练数据路径自动回退到 `backend/data/raw/training_data.csv`

## 5. 前端模块实现

## 5.1 入口、路由与状态

文件：`frontend/src/main.js`、`frontend/src/router/index.js`、`frontend/src/store/index.js`

### Axios 约定
- 请求拦截器自动挂载 JWT
- 响应拦截器直接返回 `response.data`

> 因此页面中应优先按 `response.code/response.data` 读取，而不是 `response.data.code`。

### 路由权限
- `requiresAuth`: 需登录
- `requiresAdmin`: 管理员页面（`/settings`, `/users`）
- 未登录跳转 `/login`，管理员权限不足跳回 `/dashboard`

### Vuex
- 维护 `token/user/loading/sidebarCollapsed`
- 提供登录、注册、改密、登出动作

## 5.2 页面模块职责

- `Login.vue`：登录与路由跳转

## 6. 关键数据流说明

## 6.1 预测数据流

1. 前端发起预测请求  
2. 后端读取最近数据（不足时补齐）  
3. 归一化 + 模型推理 + 反归一化  
4. 结果入库 `PredictResult`  
5. 前端读取历史并绘图

## 6.2 调度数据流

1. 前端提交调度参数和预测结果  
2. 后端组装 `DispatchModel`  
3. `AWPSO.optimize()` 求解  
4. 结果入库 `DispatchResult`  
5. 前端绘制充放电/SOC/成本指标

## 6.3 模拟执行流

1. 基于 `dispatch_id` 获取调度计划  
2. 拉取同日期真实新能源数据  
3. 计算 `grid_power`、购售电功率和实际成本  
4. 返回计划成本与实际成本偏差供前端展示
